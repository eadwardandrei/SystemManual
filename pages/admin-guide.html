OVERVIEW

Administrator Responsibilities

Administrative Overview

  • Oversee the overall operation of the TechReserve system
  • Manage and monitor user accounts across all departments
  • Review, approve, or reject reservation requests
  • Maintain accurate and up-to-date equipment records
  • Ensure fair resource allocation and operational compliance
  • Safeguard system security and administrative workflows
ACCESS CONTROL

Administrator Account Access

Administrative Authorization

  • Administrator accounts are created by system owners or super administrators
  • Access is granted only after proper verification and authorization
  • Admins receive elevated privileges for system management and configuration
  • Administrative access allows control over users, reservations, and inventory
  • Credentials must be protected to prevent unauthorized system changes
ADMIN PERMISSIONS

Administrator Capabilities

System Administrator

  • Create, update, deactivate, or delete user accounts
  • Monitor reservation logs and activity history
  • Configure system rules, limits, and operational settings
  • Generate administrative reports and summaries
  • Cancel or modify reservations when permitted

ACCOUNT MANAGEMENT

Create an Administrator Account

1

Log in to the Administrator Account

An existing administrator logs in to TechReserve using valid admin credentials to access system management features.

2

Access the Requests Hub

From the Dashboard, navigate to the Requests Hub where account-related actions and invitations are managed.

3

Open the Add Admin Account Modal

Select the Add Admin Account option to open the account creation modal for administrators.

4

Send an Admin Invitation

Using the Admin tab, enter the required information and send an invitation to the new administrator.

5

Monitor Invitation Status

The invitation appears in the View Invite Account modal within the Requests Hub, displaying its current status.

6

Invitation Accepted or Expired

If the invite is accepted, the request badge is updated to Accepted and the administrator account is added to the Manage Accounts table.

If the invite is not accepted within 7 days, it is marked as Expired and requires re-sending.

ACCOUNT MANAGEMENT

Add a Staff Account

1

Log in as Administrator

The administrator signs in to TechReserve using valid admin credentials to access administrative controls.

2

Access the Dashboard

After successful login, the administrator is redirected to the Dashboard displaying system summaries and management shortcuts.

3

Open the Manage Accounts Page

From the dashboard navigation, select Manage Accounts to view all registered user and staff records stored in the system.

4

Launch the Add Staff

Click the Add Staff button to open the staff account creation modal where new staff details can be entered.

5

Enter Staff Information

The administrator provides required staff details such as name, role, contact information, and login credentials. Submitted information is saved to the User Account Database.

6

Confirm Staff Account Creation

Upon successful submission, the newly added staff account immediately appears in the Manage Accounts table, reflecting real-time updates from the database.

7

Manage or Update Staff Records

Administrators may further update, deactivate, or delete staff accounts. Related task assignments, reservation records, and work history remain synchronized across the system databases.

RESERVATION MANAGEMENT

Manage Reservations

1

Log in to the Administrator Account

The administrator signs in to TechReserve using valid credentials to access reservation oversight features.

2

Access the Dashboard

After logging in, the administrator is redirected to the Dashboard, which provides access to all reservation-related modules.

3

Open the Pending Requests Page

The administrator navigates to the Pending Requests page where all newly submitted reservation requests are listed.

4

View Reservation Request Details

Selecting a request opens the View Request page, displaying reservation details retrieved from the reservation database.

5

Admin Verification

The administrator verifies the request for accuracy, availability, and policy compliance.

The request may be approved, sent for revision, or rejected.

6

Approve Reservation Request

Once approved, the reservation is stored in the Approved Reservation database and becomes visible on the Approved Requests page.

7

Review and Verify Workflow

The administrator reviews the reservation workflow and performs a second verification.

At this stage, the workflow may be edited, cancelled, or approved for deployment.

8

Deploy Reservation

Upon approval, the reservation is deployed and appears on the Active Reservations page for execution.

9

Process Deployment

The reservation is actively processed based on the approved schedule, assigned resources, and workflow instructions.

10

Final Admin Verification

After deployment, the administrator verifies completion.

Any issues may be flagged for reporting before final closure.

11

Confirm Return and Completion

Once the reservation is completed and confirmed, the process is officially closed within the system.

12

Archive to Past Records

Completed reservations are stored in the Past Records section and saved to the system history database for tracking and reporting.

Manage Task Assignments

1

Log in to the Administrator Account

The administrator logs in to TechReserve using valid admin credentials to access operational management tools.

2

Access the Dashboard

After authentication, the administrator is redirected to the Dashboard, which provides access to system modules.

3

Open the Task Assignments Page

From the dashboard, navigate to the Task Assignments page to manage staff task allocations.

4

View Task Assignments

The system retrieves task data from the Task Assignment Database, displaying all current assignments and assigned personnel.

5

Update Assigned Personnel

The administrator may update task assignments by selecting new personnel or reassigning existing tasks. Changes are validated and saved to the database in real time.

6

Confirm and Maintain Records

Once updates are completed, the revised task assignments are reflected immediately in the system and preserved for operational tracking and accountability.
