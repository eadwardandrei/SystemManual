Introduction Task Assignments
Platform Capability

Task Assignments Module

The Task Assignments module enables administrators to assign, track, and manage operational tasks related to reservations, equipment handling, maintenance, and system actions. It ensures accountability, workflow clarity, and timely execution.

Overview Task Types Assignment Workflow Status Tracking Task Records Notifications Admin Actions
OVERVIEW

Managing Operational Tasks

  • The Task Assignments page is used to monitor reservation-related workloads, review staff assignments, and ensure all deployments remain on schedule.
  • Provides an overview of task metrics including total assignments, in-progress tasks, completed tasks, and overdue assignments.
  • Displays a task table showing which employee or staff member is assigned to a specific reservation.
  • Allows administrators to assign operational tasks to staff members responsible for reservation handling, equipment preparation, and maintenance.
  • Integrates an SMS and notification module to immediately inform staff when a reservation or task is assigned to them.
TASK CATEGORIES

Types of Assignable Tasks

Reservation Handling

Tasks related to approving, preparing, and releasing reserved resources.

Maintenance Tasks

Assignments for inspecting, repairing, and maintaining equipment.

Equipment Turnover

Ensures proper check-in, check-out, and condition verification.

Compliance & Audit

Tasks supporting record verification and system audits.

PROCESS

Task Assignments Workflow

Identify Task

Administrator identifies the required operational task based on system activity or requests.

Assign Staff

Task is assigned to a specific staff member responsible for execution.

Set Priority & Due Date

Priority level and completion deadline are defined by the administrator.

Execute Task

Staff performs the task and updates its progress within the system.

Review & Close

Administrator reviews the outcome and marks the task as completed.

TRACKING

Task Status Monitoring

Pending

Task has been created and assigned but has not yet been started by the responsible staff member.

In Progress

Task is actively being worked on and is currently under execution.

Completed

Task has been successfully finished and verified by the administrator.

Overdue

Task has exceeded its assigned due date and requires immediate action.

Cancelled

Task was discontinued due to reservation changes or administrative decisions.

ASSIGNMENTS

Task Assignments Records

  • Displays a table listing reservations and their corresponding assigned staff or employees.
  • Shows task details such as reservation ID, task type, assigned personnel, due date, and current status.
  • Enables administrators to easily track accountability and workload distribution.
COMMUNICATION

Task Notifications

  • An SMS message and system notification are sent automatically when a task or reservation is assigned to a staff member.
  • Notifications include task details such as reservation reference, assigned responsibility, and due date.
  • Staff receive reminders for approaching deadlines or overdue tasks.
  • Administrators are notified of task completion, verification, or overdue assignments.
ADMIN CONTROLS

Managing Task Assignments

1

Select Related Reservation

The administrator identifies a reservation or system activity that requires an operational task.

2

Create Task Assignment

A task is created through the Task Assignments interface and linked to the selected reservation or resource.

3

Assign Staff Member

The administrator assigns the task to a specific staff member and defines priority and due date.

4

Send Notification

The system automatically sends a notification or SMS to inform the assigned staff of the task details.

5

Monitor Task Progress

The administrator tracks task status, progress updates, and staff activity through the task dashboard.

6

Verify and Close Task

Upon completion, the administrator reviews the task and marks it as completed or reassigns if necessary.
