Accounts Management

Accounts Management Module

The Accounts Management module allows administrators to manage user access, assign roles, and maintain system security through account verification, activation, deactivation, and permission-based controls.

Module Overview Borrower Account Create Borrower Account Update Borrower Account Admin Account Create Admin / Staff Account Verification Account Status Control
OVERVIEW

Managing System Accounts

Accounts Management Overview

  • Manages all system user accounts, including borrowers, staff, and administrators.
  • Handles account registration requests and verification through the Request Hub.
  • Controls role assignment, access permissions, and account status.
  • Allows administrators to update, deactivate, and reactivate user accounts.
  • Ensures that only authorized and verified users can access TechReserve features.
USER ACCESS

Borrower Account Management

Borrower Account Overview

  • Borrower accounts are intended for students, faculty, and authorized users.
  • These accounts allow users to request and manage reservations.
  • Borrowers have no administrative control over system data.
  • All borrower accounts require administrator approval.
STEP-BY-STEP

Creating a Borrower Account

1

Access Registration Page

The user navigates to the TechReserve registration page and selects Create Account.

2

Submit Required Information

The user provides personal and institutional details using a valid email address.

3

Account Pending Verification

The account is stored with a Pending status until reviewed by an administrator.

4

Approval and Activation

Once approved, the borrower gains access to the system dashboard.

ACCOUNT SETTINGS

Updating a Borrower Account

1

Log In

The borrower signs in using valid account credentials.

2

Open Account Settings

The user navigates to the Profile & Account Settings section.

3

Edit Information

Allowed personal details may be updated and saved.

ADMINISTRATIVE ACCESS

Administrator Account Management

Administrator Responsibilities

  • Administrators manage users, permissions, and system data.
  • They approve account requests and reservation submissions.
  • Admins control account status and role assignments.
ADMIN ACTION

Creating Admin and Staff Accounts

1

Access Accounts Management

An administrator opens the Accounts Management module.

2

Create New Account

The administrator selects Create Account and assigns a role (Administrator or Staff).

3

Assign Permissions

Role-based permissions are configured before activation.

REQUEST HUB

Account Verification

1

Review Pending Requests

Administrators access pending accounts from the Request Hub.

2

Approve or Reject

Submitted information is validated before approval or denial.

SECURITY CONTROL

Account Deactivation and Reactivation

1

Select Account

The administrator selects a user account from the database.

2

Deactivate or Reactivate

The account status is changed to Inactive or restored to Active as needed.
